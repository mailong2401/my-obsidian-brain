# 2.1 Repository là gì?

## 📌 Khái niệm

**Repository** là tầng trung gian giữa:

```
Controller ↔ Service ↔ Repository ↔ Database
```

Nhiệm vụ chính:

- Giao tiếp với database
    
- Thực hiện CRUD (Create, Read, Update, Delete)
    
- Giảm việc viết SQL thủ công
    

---

## 📌 Trong Spring Boot

Spring cung cấp:

- `JpaRepository`
    
- `CrudRepository`
    
- `PagingAndSortingRepository`
    

👉 Nhưng dùng nhiều nhất là:

```
JpaRepository
```

---

# 2.2 JpaRepository là gì?

## 📌 Định nghĩa

`JpaRepository` là interface cung cấp sẵn:

- CRUD
    
- Pagination
    
- Sorting
    
- Query methods
    

---

## 📌 Cách dùng

```java
public interface UserRepository extends JpaRepository<User, Long> {
}
```

---

## 📌 Giải thích

```java
JpaRepository<Entity, ID>
```

|Thành phần|Ý nghĩa|
|---|---|
|`User`|Entity|
|`Long`|Kiểu của khóa chính (id)|

---

# 2.3 Các method có sẵn

## 📌 CRUD cơ bản

```java
save(entity)
findById(id)
findAll()
deleteById(id)
delete(entity)
existsById(id)
count()
```

---

## 📌 Ví dụ

```java
@Autowired
private UserRepository userRepository;

// CREATE
userRepository.save(user);

// READ
User user = userRepository.findById(1L).orElse(null);

// UPDATE
user.setName("New Name");
userRepository.save(user);

// DELETE
userRepository.deleteById(1L);
```

# 2.4 Query Methods (Tự sinh query)

## 📌 Spring tự tạo query từ tên method

---

## 🔥 Ví dụ

```java
List<User> findByUsername(String username);
```

👉 SQL tương đương:

```sql
SELECT * FROM user WHERE username = ?
```

---

## 📌 Các keyword phổ biến

|Keyword|Ý nghĩa|
|---|---|
|`And`|AND|
|`Or`|OR|
|`Between`|BETWEEN|
|`LessThan`|<|
|`GreaterThan`|>|
|`Like`|LIKE|
|`Containing`|LIKE %value%|
|`StartsWith`|LIKE value%|
|`EndsWith`|LIKE %value|

---

## 🔥 Ví dụ nâng cao

```java
List<User> findByUsernameAndEmail(String username, String email);

List<User> findByAgeGreaterThan(int age);

List<User> findByNameContaining(String keyword);
```

# 2.5 @Query (Viết query thủ công)

## 📌 Khi nào dùng?

- Query phức tạp
    
- Join nhiều bảng
    
- Query custom
    

---

## 🔥 JPQL

```java
@Query("SELECT u FROM User u WHERE u.email = :email")
User findByEmail(@Param("email") String email);
```

---

## 🔥 Native SQL

```java
@Query(value = "SELECT * FROM users WHERE email = :email", nativeQuery = true)
User findByEmailNative(@Param("email") String email);
```

# 2.7 Custom Repository

---

## 📌 Khi cần logic riêng

---

### B1: Tạo interface

```java
public interface UserRepositoryCustom {
    List<User> findCustom();
}
```

---

### B2: Implement

```java
public class UserRepositoryImpl implements UserRepositoryCustom {

    @PersistenceContext
    private EntityManager entityManager;

    @Override
    public List<User> findCustom() {
        String sql = "SELECT u FROM User u";
        return entityManager.createQuery(sql, User.class).getResultList();
    }
}
```

---

### B3: Kết hợp

```java
public interface UserRepository 
    extends JpaRepository<User, Long>, UserRepositoryCustom {
}
```

# 2.8 Transaction trong Repository

## 📌 Mặc định

- `save()` → có transaction
    
- `delete()` → có transaction
    

---

## 📌 Custom

```java
@Transactional
@Modifying
@Query("UPDATE User u SET u.name = :name WHERE u.id = :id")
void updateName(@Param("id") Long id, @Param("name") String name);
```

# 2.9 Best Practices

## ✅ Nên

- Dùng `JpaRepository`
    
- Tên method rõ ràng
    
- Tách logic sang Service
    

---

## ❌ Không nên

- Viết logic business trong Repository
    
- Query quá phức tạp bằng method name
    
- Lạm dụng native SQL
    

---

# 2.10 Flow thực tế

```
Controller → Service → Repository → Database
```

---

## 🔥 Ví dụ thực tế

### Repository

```java
public interface UserRepository extends JpaRepository<User, Long> {
    Optional<User> findByUsername(String username);
}
```

---

### Service

```java
@Service
public class UserService {

    @Autowired
    private UserRepository userRepository;

    public User getUser(String username) {
        return userRepository.findByUsername(username)
                .orElseThrow(() -> new RuntimeException("User not found"));
    }
}
```
