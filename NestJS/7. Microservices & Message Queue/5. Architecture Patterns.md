
Khi áp dụng vào thực tế, có một số mô hình kiến trúc phổ biến:

### 5.1. Database per Service
Mỗi microservice cần có **CSDL riêng** (Database per Service). Các service không bao giờ truy cập trực tiếp vào database của nhau. Nếu cần dữ liệu từ service khác, hãy gọi qua API của service đó.

### 5.2. API Gateway Pattern
Tạo một service đặc biệt (API Gateway) đứng trước các microservices. Gateway nhận request từ Client (Web/Mobile), chuyển tiếp đến các service con (User Service, Product Service), tổng hợp kết quả (nếu cần) và trả về cho Client.

**Ví dụ cấu trúc thư mục (Monorepo với Nx hoặc lerna):**
```
apps/
├── api-gateway/     # (HTTP Server) - Cổng vào
├── users-service/   # (TCP Server) - Quản lý user
├── payment-service/ # (TCP/Kafka Server) - Xử lý thanh toán
libs/
└── common/          # DTOs, Interfaces dùng chung
```
*(Tham khảo các dự án như VanguardNX để thấy cách tổ chức Monorepo chuyên nghiệp)*.

### 5.3. Hybrid Application (Kết hợp HTTP + Microservices)
Bạn có thể chạy một ứng dụng vừa nhận HTTP request vừa là Microservice (hữu ích cho Gateway).

```typescript
// main.ts (Hybrid)
import { NestFactory } from '@nestjs/core';
import { AppModule } from './app.module';
import { Transport } from '@nestjs/microservices';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  
  // Kết nối thêm microservice listener
  app.connectMicroservice({
    transport: Transport.TCP,
    options: { port: 3001 },
  });

  await app.startAllMicroservices();
  await app.listen(3000); // Lắng nghe HTTP
}
bootstrap();
```
